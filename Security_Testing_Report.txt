1. SQL INJECTION

Description:
SQL Injection attack was tested on the login page by entering a malicious SQL payload in the username field.

Test Payload Used:

Testing Steps:

Navigate to login page.

Enter the payload in username field.

Enter random password.

Click Login.

Expected Result:
Application should reject invalid login credentials.

Actual Result:
The application allowed login without valid credentials and bypassed authentication.

Risk Level: High

Impact:
An attacker can gain unauthorized access to sensitive account information.
